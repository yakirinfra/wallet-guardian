\# Wallet GuardianA lightweight custody policy engine that simulates institutional wallet management flows for BILS stablecoin.\## Demo<p align="center">&#x20; <img src="assets/demo.gif" alt="Wallet Guardian Demo" width="900"></p>\---\## OverviewThis project demonstrates how financial institutions manage digital assets using a \*\*control layer before execution\*\*.Instead of sending funds directly, each transaction is evaluated against predefined security policies.\---\## Demo Flow\### 1. Wallet CreationA treasury wallet is created with an initial balance of BILS.\---\### 2. Valid Transaction — APPROVEDA transaction is sent to a whitelisted address:```json{&#x20; "to\_address": "SOL\_COMPANY\_WALLET",&#x20; "amount": 1000}```Result:```textAPPROVED```Reason:\- Address is in whitelist\- Amount is below the limit\- Wallet has sufficient balance\---\### 3. Invalid Transaction — BLOCKEDA transaction is sent to an unknown wallet address:```json{&#x20; "to\_address": "UNKNOWN\_WALLET",&#x20; "amount": 1000}```Result:```textBLOCKED```Reason:\- Destination address is not whitelisted\---\### 4. High Amount Transaction — PENDING\_APPROVALA transaction is sent to a whitelisted address, but the amount is above the allowed limit:```json{&#x20; "to\_address": "SOL\_COMPANY\_WALLET",&#x20; "amount": 8000}```Result:```textPENDING\_APPROVAL```Reason:\- Amount exceeds the maximum single transaction limit of 5000\---\## Features\- Simulated wallet creation\- Policy-based transaction validation\- Whitelist enforcement\- Balance validation\- Transaction limits\- Approval flow simulation\- Audit logging\---\## Architecture```textClient → FastAPI → Policy Engine → Decision → Audit Log```\---\## Tech Stack\- Python\- FastAPI\- Uvicorn\---\## Run Locally```bashpip install -r requirements.txtuvicorn main:app --reload```Open Swagger UI:```texthttp://127.0.0.1:8000/docs```\---\## Key ConceptThis project simulates a \*\*custody control layer\*\* where:\- Transactions are not executed directly\- Each request is validated against security policies\- Risky transactions require manual approval\- Every action is auditable\---\## Future Improvements\- Multi-signature approval flow\- Real blockchain integration with Solana or Ethereum\- Persistent database with PostgreSQL\- Role-based access control\- Real-time alerts\- Docker deployment
